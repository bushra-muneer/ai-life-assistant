# Day 2 Notes

Day 2 focuses on the app shell and module navigation.

## Product decision

The MVP interface stays English-first. This keeps the first version clean, premium, and easier to test.

The product is still designed for Urdu, Hindi, and Roman Urdu users. Multilingual assistant responses and a language preference can come after the core flow works.

## Added in this step

- Expanded module metadata
- How it works section
- Roadmap preview section
- Responsive shell styles

## Next product question

Which module should become interactive first?

Recommended answer: Daily Life Assistant, because it is the broadest use case and will validate the core chat flow fastest.
