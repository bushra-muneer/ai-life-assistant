# Working Style

This repo should stay clear and easy to review.

## Workflow

- Build one small improvement at a time.
- Keep each branch focused.
- Use readable file and component names.
- Write notes for important product choices.
- Test the basic flow before adding extra features.

## Code style

- Keep components small.
- Keep copy simple.
- Prefer simple CSS while the app is small.
- Move shared data into separate files when it helps readability.
